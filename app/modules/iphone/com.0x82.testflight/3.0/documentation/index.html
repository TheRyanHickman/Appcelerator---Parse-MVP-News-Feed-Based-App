testflight Module

Description

This Titanium module wraps the oficial Testflight SDK, available at https://testflightapp.com/sdk/.

It allows you to write Titanium apps and still benefict from using the Testflight SDK.

Accessing the testflight Module

To access this module from JavaScript, you would do the following:

var testflight = require("com.0x82.testflight");

The testflight variable is a reference to the Module object.

Reference

testflight.takeOff(appToken, [options])

This should be the first function you should call to initialize the module. It starts a TestFlight session, setups the crash handler, and sends pending notifications to the server.

It no longer accepts a Team Token

It accepts a single argument appToken. This will be the application token for the current application. The token for this application can be retrived by going to https://testflightapp.com/dashboard/applications/ .

It also accepts an optional object that allows you to define advanced options:

  • testflight.DISABLE_IN_APP_UPDATES - the in application update screen will not be displayed. Defaults to false.
  • testflight.LOG_ON_CHECKPOINT - Because logging is synchronous, if you have a high preformance app, you might want to turn this off. Defauts to true.
  • testflight.LOG_TO_CONSOLE - Prints remote logs to Apple System Log. Defaults to true.
  • testflight.LOG_TO_STDERR - Sends remote logs to STDERR when debugger is attached. Defaults to true.
  • testflight.REPORT_CRASHES - If set to false, crash handlers are never installed. Defaults to true.
  • testflight.SEND_LOG_ONLY_ON_CRASH - Setting to true stops remote logs from being sent when sessions end. They would only be sent in the event of a crash. Defaults to false.
  • testflight.SESSION_KEEP_ALIVE_TIMEOUT - This is the amount of time a user can leave the app for and still continue the same session when they come back. If they are away from the app for longer, a new session is created when they come back. Must be a number. Change to 0 to turn off. Defaults to 30.
  • testflight.FLUSH_SECONDS_INTERVAL - Defaults to 60. Set to a number. 0 turns off the flush timer. 30 seconds is the minimum flush interval.

Example:

var options = {};
options[testflight.DISABLE_IN_APP_UPDATES] = false;

testflight.takeOff('token', options);

testflight.passCheckpoint(checkPointName)

Track when a user has passed a checkppint after the flight has taken off. Eg. passed level 1, posted hight score, etc.

It accepts a single argument checkPointName with the String name of the checkpoint.

testflight.addCustomEnvironmentInformation(hashOptions)

Add custom environment information. If you want to track a user name from your application you can add it here.

It accepts a single hash/dictionary argument hashOptions, with key/value pairs of options you want to set on your environment.

testflight.submitFeedback(text)

Sends a custom feedback to the server without showing the feedback view. Accepts a single parameter which should be a String with the text to send to the testflight server.
